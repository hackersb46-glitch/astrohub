# AstroHub 产品规格 SPEC.md

## 页面布局规则
- 一级菜单：横向布局（顶部导航栏）
- 二级菜单：画面左侧展开
- 进入一级菜单后，根据是否有二级菜单选择显示

## 设备连接规则
- 连接后可根据搜索到的设备选择
- 一旦连接，后续默认使用当前连接设备，无需重复选择
- 设备列表显示：设备型号 + 设备名称 + IP 地址（不是 IP(IP) 方式）
- 同一物理设备只显示一条记录，MAC 和 IP 合并为单一实体

## 一级菜单结构

### 1. 仪表盘
- **本机信息**（可折叠）：CPU型号、内存、GPU、网络占用、IP地址等
- **已连接设备信息**（可折叠）：设备名、IP、MAC、型号、固件版本、PTZ位置
- **状态指示**：推流开关状态、录像开关状态、定时截图开关状态
- **连接时长**：已连接时间计时器
- **最近10次操作**：人工操作历史记录

### 2. 设备
- **左右结构**
- **左侧**：设备连接信息（SADP 扫描、修改 IP、登录账号密码）
  - 单选按钮：一体式 PTZ 设备（网络）/ 串口设备 / USB 设备
  - 未选择时下方灰色不可交互，选择后可交互
  - 目前只有 PTZ 已开发，串口/USB 占位
- **右侧上方**：设备实时画面（HTTP 取流）
- **右下**：程序运行 print 信息

### 3. 主控台
- **左侧**：指向、跟踪开关、截图、录像、推流开关、画面调节、摇杆控制 PTZ、对焦
- **右侧**：设备实时画面

### 3.1 ASCOM Alpaca Server（AstroHub 作为 ASCOM 设备提供方）
- **核心逻辑**：AstroHub 启动后运行 ASCOM Alpaca Server，将本机 PTZ 硬件映射为 ASCOM 设备，供外部软件发现和使用
- **不是**连接外部 ASCOM 设备，而是**输出** ASCOM 设备
- **设备映射**：
  - **PTZ Pan/Tilt → ASCOM Telescope**：映射为望远镜/赤道仪，支持 Slew（指向）、Track（跟踪）、坐标读取
  - **焦点控制 → ASCOM Focuser**：映射为调焦器，支持位置设置、移动、状态查询
  - **日夜切换 → ASCOM Filter Wheel**：映射为滤镜轮（日间/夜间/IR 切换）
- **Alpaca Server 端口**：5555（标准）
- **设备发现端点**：`http://localhost:5555/management/v1/apidevices`
- **外部软件**：NINA、ASCOM Platform、SkySafari 等可通过 Alpaca 协议发现并连接 AstroHub 提供的设备
- **进入条件**：新手引导全部完成后（功能探测 → 限位测试 → 速度测试 → Goto Home），设备进入主控台

### 4. 观测计划
- **左侧**：目标推荐（太阳系天体、深空天体）
- **右侧**：观测计划（多目标序列、定时开关）
- **右下**：启动/停止按钮，前后选项

### 5. 高级功能
- **左侧二级菜单**：A校准 B功能测试 C限位测试 D速度测试 E推流/存储设置
- **页面左右布局**：左上操作区、左下实时文本信息、右侧当前画面
- 所有改动集中在对应二级菜单页面内，不新增额外一级菜单

### 6. 回放
- 图像、视频，根据观测目标浏览

## 新手引导（新设备首次连接）
1. 发现设备后，弹出引导向导
2. 引导用户完成：功能探测 → 限位测试 → 速度测试（严格按此顺序）
3. 全部完成后，设备进入主控台

## 网络配置
- AstroHub 服务默认端口: **10280** (环境变量: `ASTROHUB_PORT`)
- ASCOM Alpaca Server 端口: **5555** (标准)
