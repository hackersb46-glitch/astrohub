## 算法开发方法总结

### 1. 图像叠加算法

**位置**: `documents/ALGORITHM_DEVELOPMENT.md`

**核心组件**:
- 星点检测: photutils.DAOStarFinder (推荐) / OpenCV (降级)
- 图像对齐: astroalign (XY+旋转) / 相位相关 (仅平移)
- 叠加算法: Sigma Clipping (后处理) / Running Mean (实时)

**关键教训**:
- Live Stack 对齐必须包含旋转
- Running Mean 是 O(1) 内存，Sigma Clipping 是 O(N帧)
- FITS Header 必填: EXPTIME, NFRAMES, DATE-OBS, STACKMETH

### 2. PTZ 跟踪算法

**三种场景**:
| 场景 | 输入 | 输出 |
|------|------|------|
| 恒星跟踪 | RA/DEC + 时间 + 纬度 | Pan/Tilt 速度 |
| 图像引导跟踪 | 对齐变换历史 | 漂移速度 |
| 指向跟踪 | 当前坐标 + 目标坐标 | 移动指令 |

**计算方法**:
- 恒星时角变化率: 15°/小时 (常数)
- 图像漂移: 从 astroalign 变换历史计算
- 速度映射: 归一化到 PTZ 档位 (1-100)

### 3. PTZ 指向算法

**流程**:
```
RA/DEC → Alt/Az → T-Point 修正 → PTZ 坐标 → 绝对移动
```

**关键组件**:
- 坐标转换: astropy.coordinates
- T-Point 模型: 6 参数 (IA, IH, NP, level_ns, level_ew, AW)
- PTZ 映射: Pan=az×10, Tilt=alt×10 (0.1° 分辨率)

### 4. 天文校准算法

**两种方案**:
| 方案 | 依赖 | 精度 |
|------|------|------|
| 手动对准 | 无 | 用户决定 |
| 自动 Plate Solve | ASTAP | 亚像素级 |

**校准步骤**:
1. 选择 ≥10 颗亮星（均匀分布）
2. PTZ 指向 → 对准中心 → 记录坐标
3. 拟合 T-Point 参数（最小二乘）
