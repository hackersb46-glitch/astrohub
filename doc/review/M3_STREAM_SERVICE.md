# M3 流媒体服务

## 概述
视频流拉取、转码、分发、画面预览。

## 前置约束
> 版本: v1.0
> 文档用途: 每次运行后逐项自检，达成标准则该项通过，未达成则记录问题
> 开发过程中，前一个P没有通过，不得跳过当前P，必须修复直到通过评审点
> 不允许做任何假设，如有问题，就是评审点不通过
> 任何有明确返回错误代码的，都需要查询错误码具体是什么
> 必须全量搜索已有案例、查询官方文档，解决问题
> 遇到阻滞，第一时间回看官方文档，查找关键词，全量阅读。禁止向人类发问
> 每一个P自检通过评审之后，将正确方法记录在对应method.md表格的对应操作方法中

## 评审明细

| 编号 | 标题 | 内容描述 | 操作方法 | 评审标准 |
|------|------|---------|---------|---------|
| P0 | 视频流接入 | RTSP/ONVIF拉流、流地址解析、认证接入 | | |
| P0.1 | RTSP地址解析 | 解析RTSP流地址，提取host/port/path/认证信息 | 输入完整RTSP URL，输出结构化字段（协议/用户名/密码/host/port/path） | 标准格式rtsp://user:pass@host:port/path解析正确；缺省端口使用默认554；非法格式返回解析错误 |
| P0.2 | ONVIF服务发现 | 通过ONVIF WS-Discovery发现设备流地址 | 发送UDP组播发现消息，解析设备返回的XAddr和Profile信息 | 10秒内返回设备列表；包含设备XAddr地址和可用Profile；无设备时返回空列表 |
| P0.3 | 流认证接入 | 使用设备凭证建立流连接 | 使用ISAPI或ONVIF认证获取流token/URL | 认证成功获得有效流地址；认证失败(401)返回明确错误并允许重试；凭证过期自动刷新 |
| P0.4 | 多协议适配 | 支持RTSP/ONVIF/HTTP-FLV多种拉流协议 | 根据设备类型和配置自动选择最优拉流协议 | 各协议均能成功拉流；协议切换时流不中断或平滑过渡 |
| P1 | 流媒体转码 | H264/H265转码、分辨率适配、码率控制 | | |
| P1.1 | 编码格式检测 | 检测输入流的编码格式 | 解析流头部信息，识别H264/H265/MJPEG等编码 | 正确识别编码格式；未知编码格式返回明确错误 |
| P1.2 | H264/H265转码 | 将输入流转码为指定编码格式 | 使用FFmpeg或硬件编码器进行实时转码 | 转码后流可在目标播放器正常播放；转码延迟<2秒；CPU占用在可接受范围 |
| P1.3 | 分辨率适配 | 将流调整为指定分辨率 | 设置输出分辨率（1080p/720p/480p），执行实时缩放 | 输出流分辨率与设定一致；画面比例正确无变形；缩放后画质可接受 |
| P1.4 | 码率控制 | 控制输出流的码率 | 设置目标码率（如2Mbps/4Mbps），启用CBR/VBR模式 | 实际输出码率在目标值±10%范围内；码率切换平滑无卡顿 |
| P2 | 流分发 | WebSocket推流、HTTP-FLV、HLS、多路并发 | | |
| P2.1 | WebSocket推流 | 通过WebSocket向客户端推送视频帧 | 建立WebSocket连接，按帧推送H264/H265 NALU数据 | 客户端可接收并解码播放；连接断开后自动清理资源 |
| P2.2 | HTTP-FLV分发 | 提供HTTP-FLV流地址供播放器拉取 | 启动HTTP-FLV服务器，监听指定端口，提供.flv流地址 | 播放器可通过http://host:port/stream.flv正常播放；多客户端同时拉取不冲突 |
| P2.3 | HLS分发 | 生成HLS m3u8+ts切片供Web播放 | 生成m3u8播放列表和ts切片文件，设置切片时长 | 浏览器可通过HLS播放器正常播放；切片生成延迟<3秒；旧切片自动清理 |
| P2.4 | 多路并发 | 同时处理多路视频流 | 配置并发路数（默认4路），同时拉取/转码/分发 | 并发路数达到配置值时仍能正常工作；超过限制时排队或拒绝并返回明确提示 |
| P3 | 画面预览 | Web播放器集成、延迟优化、截图功能 | | |
| P3.1 | Web播放器集成 | 在Web页面中嵌入视频播放器 | 集成flv.js/hls.js播放器，配置流地址 | 页面加载后自动播放；播放器控件完整（播放/暂停/全屏/音量） |
| P3.2 | 延迟优化 | 优化端到端延迟 | 调整缓冲区大小、关键帧间隔、传输协议 | 端到端延迟<1秒（同网络）；延迟数据可通过API查询 |
| P3.3 | 截图功能 | 从视频流中截取当前帧为图片 | 发送截图命令，保存为JPEG/PNG文件 | 截图文件可正常打开；画面与播放画面一致；截图响应时间<2秒 |
| P4 | 流状态监控 | 断流检测、自动重连、状态上报 | | |
| P4.1 | 断流检测 | 检测视频流是否中断 | 监控流数据到达间隔，超过阈值（默认10秒）判定断流 | 断流后5秒内检测到并标记流状态为disconnected；误判率<1% |
| P4.2 | 自动重连 | 断流后自动重新连接 | 检测到断流后，按重试策略（间隔/次数）重新拉流 | 重试成功后流恢复正常；达到最大重试次数后标记失败并通知 |
| P4.3 | 状态上报 | 定期上报流状态信息 | 上报内容包括：流状态/码率/分辨率/帧率/延迟/客户端数 | 上报数据准确；上报间隔可配置（默认30秒）；状态变化时立即上报 |
| P5 | 录制存储 | 本地录制、云端上传、录像检索 | | |
| P5.1 | 本地录制 | 将视频流录制到本地文件 | 启动录制后持续写入MP4/FLV文件，支持按时间或大小分段 | 录制文件可正常播放；文件大小与录制时长匹配；分段文件连续无丢失 |
| P5.2 | 录制控制 | 启动/停止/暂停录制 | 提供API控制录制状态 | 启动后立即开始录制；停止后文件正确关闭；暂停时不写入数据 |
| P5.3 | 录像检索 | 按时间/设备检索已录制文件 | 查询录制文件索引，返回匹配的文件列表 | 检索结果准确；支持时间范围过滤；返回文件包含时长/大小/设备信息 |
