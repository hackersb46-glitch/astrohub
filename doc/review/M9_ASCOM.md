# M9 ASCOM 集成

## 概述
ASCOM天文设备集成，望远镜、相机、焦点器控制。

## 前置约束
> 版本: v1.0
> 文档用途: 每次运行后逐项自检，达成标准则该项通过，未达成则记录问题
> 开发过程中，前一个P没有通过，不得跳过当前P，必须修复直到通过评审点
> 不允许做任何假设，如有问题，就是评审点不通过
> 任何有明确返回错误代码的，都需要查询错误码具体是什么
> 必须全量搜索已有案例、查询官方文档，解决问题
> 遇到阻滞，第一时间回看官方文档，查找关键词，全量阅读。禁止向人类发问
> 每一个P自检通过评审之后，将正确方法记录在对应method.md表格的对应操作方法中

## 评审明细

| 编号 | 标题 | 内容描述 | 操作方法 | 评审标准 |
|------|------|---------|---------|---------|
| P0 | ASCOM环境 | ASCOM Platform安装、驱动注册、连接测试 | | |
| P0.1 | ASCOM Platform检测 | 检测系统是否安装ASCOM Platform | 查询注册表或文件系统确认ASCOM Platform安装 | 已安装返回版本信息；未安装返回明确错误和安装指引 |
| P0.2 | 驱动注册 | 注册ASCOM驱动到系统 | 将驱动DLL注册到ASCOM驱动管理器 | 注册成功；驱动在ASCOM Chooser中可见 |
| P0.3 | 连接测试 | 测试与ASCOM设备的连接 | 通过ASCOM API连接设备，验证通信 | 连接成功返回设备信息；连接失败返回错误码和描述 |
| P1 | 望远镜控制 | 望远镜连接、Slew控制、位置查询、跟踪模式 | | |
| P1.1 | 望远镜连接 | 连接ASCOM望远镜驱动 | 通过ASCOM Telescope接口连接 | 连接成功；可获取望远镜状态（是否homing/是否tracking） |
| P1.2 | Slew控制 | 控制望远镜Slew到目标坐标 | 发送目标赤经/赤纬，执行SlewToCoordinates | Slew成功到达目标位置；位置误差<1角分 |
| P1.3 | 位置查询 | 查询望远镜当前位置 | 读取RightAscension和Declination属性 | 返回坐标准确；与望远镜控制器显示一致 |
| P1.4 | 跟踪模式 | 设置望远镜跟踪模式 | 设置Tracking模式（sidereal/lunar/solar/off） | 跟踪模式正确切换；跟踪精度可接受 |
| P1.5 | Slew取消 | 取消正在进行的Slew操作 | 调用AbortSlew | Slew立即停止；望远镜停在当前位置 |
| P2 | 相机控制 | 相机连接、曝光控制、图像获取 | | |
| P2.1 | 相机连接 | 连接ASCOM相机驱动 | 通过ASCOM Camera接口连接 | 连接成功；可获取相机信息（分辨率/像素大小/支持的binning） |
| P2.2 | 曝光控制 | 设置曝光参数并拍摄 | 设置曝光时间/ROI/binning，调用StartExposure | 曝光成功完成；图像数据可获取 |
| P2.3 | 图像获取 | 获取拍摄的图像数据 | 调用ImageArray获取像素数据 | 图像数据完整；格式正确（二维数组） |
| P2.4 | 相机设置 | 设置相机参数（温度/gain/offset） | 设置Camera参数 | 参数设置生效；读取返回值与设定值一致 |
| P3 | 焦点器控制 | 焦点器连接、步进控制、自动对焦 | | |
| P3.1 | 焦点器连接 | 连接ASCOM焦点器驱动 | 通过ASCOM Focuser接口连接 | 连接成功；可获取焦点器信息（位置/最大步数/温度补偿） |
| P3.2 | 步进控制 | 控制焦点器移动 | 调用Move方法移动到指定位置 | 移动成功到达目标位置；位置准确 |
| P3.3 | 自动对焦 | 实现自动对焦功能 | 通过HFD（半高全宽）测量找到最佳焦点位置 | 自动对焦成功；HFD达到最小值 |
| P3.4 | 温度补偿 | 焦点器温度补偿 | 启用温度补偿功能 | 温度变化时焦点位置自动调整；补偿量正确 |
