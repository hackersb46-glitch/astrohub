# M5 数据库管理

## 概述
设备数据、观测数据、配置数据的存储与查询。

## 前置约束
> 版本: v1.0
> 文档用途: 每次运行后逐项自检，达成标准则该项通过，未达成则记录问题
> 开发过程中，前一个P没有通过，不得跳过当前P，必须修复直到通过评审点
> 不允许做任何假设，如有问题，就是评审点不通过
> 任何有明确返回错误代码的，都需要查询错误码具体是什么
> 必须全量搜索已有案例、查询官方文档，解决问题
> 遇到阻滞，第一时间回看官方文档，查找关键词，全量阅读。禁止向人类发问
> 每一个P自检通过评审之后，将正确方法记录在对应method.md表格的对应操作方法中

## 评审明细

| 编号 | 标题 | 内容描述 | 操作方法 | 评审标准 |
|------|------|---------|---------|---------|
| P0 | 数据库初始化 | 数据库创建、表结构初始化、迁移管理 | | |
| P0.1 | 数据库创建 | 创建或连接数据库 | 检查数据库是否已存在，不存在则创建；存在则验证连接 | 数据库创建成功或已有数据库可正常连接；连接字符串正确配置 |
| P0.2 | 表结构初始化 | 创建所有必需的数据表 | 根据schema.sql或ORM定义创建表结构，包括索引和约束 | 所有表创建成功；索引和约束正确应用；重复执行不报错（幂等） |
| P0.3 | 数据库迁移 | 管理数据库schema版本升级 | 使用迁移脚本或工具（如Alembic）管理schema变更 | 迁移可正向执行和回滚；迁移记录版本号；迁移失败时回滚不破坏数据 |
| P0.4 | 连接池管理 | 管理数据库连接池 | 配置连接池大小、超时、空闲回收策略 | 连接池正常工作；连接数在配置范围内；空闲连接自动回收 |
| P1 | 设备数据存储 | 设备信息CRUD、状态记录、配置快照 | | |
| P1.1 | 设备信息写入 | 将设备信息写入数据库 | 接收设备数据（MAC/IP/型号/状态等），执行INSERT操作 | 写入后查询确认数据完整；字段类型和约束正确生效 |
| P1.2 | 设备信息读取 | 从数据库读取设备信息 | 支持按MAC精确查询、按条件过滤查询、批量查询 | 读取数据与写入数据完全一致；查询性能可接受（单条<100ms） |
| P1.3 | 设备信息更新 | 更新设备信息 | 以MAC为键更新指定字段 | 更新后查询确认变更生效；不存在的MAC返回明确错误 |
| P1.4 | 设备信息删除 | 从数据库删除设备信息 | 以MAC为键删除设备记录 | 删除后查询确认数据不存在；级联删除关联数据（配置/日志等） |
| P1.5 | 状态记录 | 记录设备状态变更历史 | 每次状态变更插入状态记录（时间戳/旧状态/新状态） | 状态记录完整；支持按时间范围和设备查询 |
| P1.6 | 配置快照 | 保存设备配置快照 | 在配置变更时保存完整配置快照 | 快照包含完整配置数据；可按时间回溯 |
| P2 | 观测数据存储 | 观测记录存储、CSV数据入库、时序数据管理 | | |
| P2.1 | 观测记录存储 | 存储观测事件数据 | 接收观测数据（时间/设备/坐标/参数），批量写入 | 写入后数据可查询；批量写入性能可接受（1000条<5秒） |
| P2.2 | CSV数据入库 | 从CSV文件批量导入观测数据 | 解析CSV文件，验证数据格式，批量写入数据库 | 导入成功数/失败数正确；非法行跳过并记录；大文件（10万行）导入不超时 |
| P2.3 | 时序数据管理 | 管理时序数据的存储和清理 | 按时间分区存储，支持自动清理过期数据 | 过期数据按策略清理；清理不影响当前数据；清理操作记录日志 |
| P3 | 查询与索引 | 查询性能优化、索引管理、聚合查询 | | |
| P3.1 | 查询性能优化 | 确保常用查询在可接受时间内完成 | 为常用查询字段创建索引，测试查询性能 | 常用查询响应时间<100ms；索引覆盖所有高频查询 |
| P3.2 | 索引管理 | 创建和维护数据库索引 | 根据查询模式创建索引，定期分析索引使用情况 | 索引正确使用；无用索引可识别和清理；索引不影响写入性能过多 |
| P3.3 | 聚合查询 | 支持统计聚合查询 | 实现按设备/时间范围的统计查询（计数/平均/最大/最小） | 聚合结果正确；大数据量（10万条）聚合<5秒 |
| P4 | 备份与恢复 | 数据库备份、恢复、一致性检查 | | |
| P4.1 | 数据库备份 | 定期备份数据库 | 导出数据库为SQL dump或二进制备份文件 | 备份文件完整；备份过程不阻塞正常操作；备份文件名包含时间戳 |
| P4.2 | 数据库恢复 | 从备份恢复数据库 | 读取备份文件，恢复到数据库 | 恢复后数据与备份时完全一致；恢复过程可中断并回滚 |
| P4.3 | 一致性检查 | 检查数据库数据一致性 | 运行一致性检查脚本，验证外键约束、数据完整性 | 检查通过无错误；发现问题时输出详细报告 |
