# M8 WebSocket 实时通信

## 概述
设备状态实时推送、流数据实时传输、命令实时下发。

## 前置约束
> 版本: v1.0
> 文档用途: 每次运行后逐项自检，达成标准则该项通过，未达成则记录问题
> 开发过程中，前一个P没有通过，不得跳过当前P，必须修复直到通过评审点
> 不允许做任何假设，如有问题，就是评审点不通过
> 任何有明确返回错误代码的，都需要查询错误码具体是什么
> 必须全量搜索已有案例、查询官方文档，解决问题
> 遇到阻滞，第一时间回看官方文档，查找关键词，全量阅读。禁止向人类发问
> 每一个P自检通过评审之后，将正确方法记录在对应method.md表格的对应操作方法中

## 评审明细

| 编号 | 标题 | 内容描述 | 操作方法 | 评审标准 |
|------|------|---------|---------|---------|
| P0 | WebSocket服务器 | 连接管理、消息路由、心跳机制 | | |
| P0.1 | WS连接建立 | 客户端建立WebSocket连接 | 客户端连接/ws端点，完成握手 | 连接成功建立；连接ID唯一；握手时间<1秒 |
| P0.2 | 消息路由 | 根据消息类型路由到对应处理器 | 定义消息格式{type, payload}，按type路由 | 每种消息类型路由到正确处理器；未知类型返回错误 |
| P0.3 | 心跳机制 | 保持连接活跃，检测死连接 | 定期发送ping，客户端回复pong；超时未回复关闭连接 | 心跳间隔可配置（默认30秒）；超时（3次无pong）关闭连接 |
| P0.4 | 连接认证 | WS连接需要认证 | 连接时携带token，服务端验证 | 有效token允许连接；无效/过期token拒绝连接 |
| P0.5 | 连接关闭 | 正常和异常关闭处理 | 处理客户端主动关闭、网络断开、服务端关闭 | 关闭后清理资源；连接状态正确更新 |
| P1 | 设备状态推送 | 状态变更推送、在线状态广播、异常通知 | | |
| P1.1 | 状态变更推送 | 设备状态变化时推送给订阅客户端 | 设备状态变更时，向订阅该设备的客户端推送消息 | 推送延迟<1秒；消息格式正确（设备MAC/新状态/时间戳） |
| P1.2 | 在线状态广播 | 定期广播所有设备在线状态 | 每隔N秒向所有客户端广播设备在线状态汇总 | 广播数据准确；客户端可据此更新UI |
| P1.3 | 异常通知 | 设备异常时推送通知 | 设备发生异常（断流/认证失败/离线）时推送 | 异常通知包含设备MAC/异常类型/时间/建议操作 |
| P2 | 流数据推送 | 视频帧推送、音频流推送、实时预览 | | |
| P2.1 | 视频帧推送 | 通过WS推送视频帧 | 将视频帧编码后通过WS推送给客户端 | 客户端可接收并解码播放；帧率可接受（>15fps） |
| P2.2 | 音频流推送 | 通过WS推送音频数据 | 将音频数据通过WS推送 | 音频播放正常；无爆音/断续 |
| P2.3 | 流推送控制 | 客户端可控制流推送的启停 | 客户端发送start/stop消息控制流推送 | 控制命令立即生效；停止后不再推送数据 |
| P3 | 命令下发 | PTZ控制指令、参数设置、批量操作 | | |
| P3.1 | PTZ指令下发 | 通过WS发送PTZ控制命令 | 客户端发送PTZ指令（pan/tilt/zoom），服务端转发到设备 | 指令成功发送到设备；设备响应返回给客户端 |
| P3.2 | 参数设置 | 通过WS设置设备参数 | 客户端发送参数设置命令 | 参数设置成功；设置结果返回给客户端 |
| P3.3 | 批量命令 | 向多个设备发送命令 | 客户端发送批量命令，服务端逐台执行 | 返回执行汇总（成功数/失败数/失败详情） |
| P4 | 连接管理 | 断线重连、并发控制、连接池 | | |
| P4.1 | 断线重连 | 客户端断线后自动重连 | 客户端实现重连逻辑，服务端支持重连恢复 | 重连后恢复订阅状态；重连次数有限制（默认5次） |
| P4.2 | 并发控制 | 限制单客户端最大连接数 | 同一token/IP最大连接数限制 | 超过限制时拒绝新连接并返回明确提示 |
| P4.3 | 连接池 | 服务端管理活跃连接池 | 维护活跃连接列表，支持查询和管理 | 可查询当前连接数和详情；连接池信息准确 |
| P4.4 | 连接超时清理 | 清理超时不活跃连接 | 连接超过超时时间无消息则关闭 | 超时时间可配置（默认5分钟）；清理操作记录日志 |
