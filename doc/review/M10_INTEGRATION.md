# M10 系统集成

## 概述
各模块集成测试、端到端流程验证、性能优化。

## 前置约束
> 版本: v1.0
> 文档用途: 每次运行后逐项自检，达成标准则该项通过，未达成则记录问题
> 开发过程中，前一个P没有通过，不得跳过当前P，必须修复直到通过评审点
> 不允许做任何假设，如有问题，就是评审点不通过
> 任何有明确返回错误代码的，都需要查询错误码具体是什么
> 必须全量搜索已有案例、查询官方文档，解决问题
> 遇到阻滞，第一时间回看官方文档，查找关键词，全量阅读。禁止向人类发问
> 每一个P自检通过评审之后，将正确方法记录在对应method.md表格的对应操作方法中

## 评审明细

| 编号 | 标题 | 内容描述 | 操作方法 | 评审标准 |
|------|------|---------|---------|---------|
| P0 | 集成环境搭建 | 全模块部署、配置整合、服务启动 | | |
| P0.1 | 全模块部署 | 部署所有模块到测试环境 | 启动后端API/前端/流媒体/数据库/ASCOM服务 | 所有服务启动成功；健康检查通过 |
| P0.2 | 配置整合 | 整合各模块配置文件 | 统一配置管理，确保模块间配置一致 | 配置加载无错误；跨模块配置一致（如数据库连接/设备凭证） |
| P0.3 | 服务启动验证 | 验证所有服务可正常通信 | 通过API调用验证模块间通信 | 模块间API调用成功；无连接错误 |
| P0.4 | 测试数据准备 | 准备集成测试所需的测试数据 | 插入测试设备/配置/观测数据 | 测试数据完整；可用于端到端测试 |
| P1 | 端到端流程 | 设备发现→认证→流预览→校准完整流程 | | |
| P1.1 | 设备发现→认证 | 完整设备发现并认证流程 | 从设备发现开始，到完成认证和凭证保存 | 全流程无错误；设备状态为active |
| P1.2 | 流预览流程 | 从设备到流预览的完整流程 | 启动设备→拉流→转码→Web预览 | 视频在Web页面正常播放；端到端延迟<2秒 |
| P1.3 | 校准流程 | 从设备到校准完成的完整流程 | 选择设备→启动校准→完成校准→查看报告 | 校准完成；报告生成；设备参数恢复到校准前 |
| P1.4 | 全流程整合 | 设备发现→认证→流预览→校准的完整流程 | 从零基础到完整使用所有功能 | 全流程无错误；各环节数据正确传递 |
| P2 | 性能测试 | 并发设备数、流延迟、API响应时间 | | |
| P2.1 | 并发设备测试 | 测试系统可管理的最大设备数 | 同时管理N台设备，执行发现/监控/配置操作 | 达到目标设备数（如10台）时系统正常工作；超出时优雅降级 |
| P2.2 | 流延迟测试 | 测量端到端流延迟 | 从设备到Web播放器的完整延迟测量 | 同网络延迟<1秒；广域网延迟<3秒 |
| P2.3 | API响应测试 | 测量API响应时间 | 对所有API端点执行压力测试 | P95响应时间<500ms；并发100请求时P99<2秒 |
| P3 | 异常恢复 | 断流恢复、设备离线恢复、服务重启 | | |
| P3.1 | 断流恢复 | 视频流中断后的自动恢复 | 模拟网络中断，验证自动重连 | 断流后30秒内自动恢复；恢复后流正常播放 |
| P3.2 | 设备离线恢复 | 设备离线后重新上线的恢复 | 模拟设备离线（断开网络），验证恢复 | 设备重新上线后自动检测并恢复连接 |
| P3.3 | 服务重启恢复 | 服务重启后状态恢复 | 重启后端服务，验证状态恢复 | 重启后设备状态/流状态/配置正确恢复；无数据丢失 |
| P4 | 集成报告 | 测试结果汇总、问题清单、修复建议 | | |
| P4.1 | 测试结果汇总 | 生成集成测试结果汇总报告 | 汇总所有测试项的通过/失败状态 | 报告完整包含所有测试项；通过率和失败详情清晰 |
| P4.2 | 问题清单 | 列出所有发现的问题 | 记录每个问题的描述、严重程度、复现步骤 | 问题清单完整；严重程度分级正确 |
| P4.3 | 修复建议 | 针对问题提供修复建议 | 分析每个问题的根因，提供修复方案 | 建议具体可操作；包含优先级 |
