# M7 REST API

## 概述
FastAPI后端，提供设备管理、流控制、校准、数据查询API。

## 前置约束
> 版本: v1.0
> 文档用途: 每次运行后逐项自检，达成标准则该项通过，未达成则记录问题
> 开发过程中，前一个P没有通过，不得跳过当前P，必须修复直到通过评审点
> 不允许做任何假设，如有问题，就是评审点不通过
> 任何有明确返回错误代码的，都需要查询错误码具体是什么
> 必须全量搜索已有案例、查询官方文档，解决问题
> 遇到阻滞，第一时间回看官方文档，查找关键词，全量阅读。禁止向人类发问
> 每一个P自检通过评审之后，将正确方法记录在对应method.md表格的对应操作方法中

## 评审明细

| 编号 | 标题 | 内容描述 | 操作方法 | 评审标准 |
|------|------|---------|---------|---------|
| P0 | 项目初始化 | FastAPI项目创建、依赖安装、目录结构 | | |
| P0.1 | 项目创建 | 使用FastAPI创建后端项目 | 创建main.py，配置app实例，设置CORS中间件 | 项目可启动；访问/docs显示OpenAPI文档；CORS配置正确 |
| P0.2 | 依赖安装 | 安装所需Python包 | 安装fastapi/uvicorn/pydantic/sqlalchemy等依赖 | 所有依赖安装成功；requirements.txt记录所有依赖 |
| P0.3 | 目录结构 | 建立标准后端目录结构 | 创建api/models/schemas/services/middleware等目录 | 目录结构清晰；符合FastAPI最佳实践 |
| P1 | 设备API | 设备CRUD、状态查询、配置管理API | | |
| P1.1 | 创建设备API | POST /api/devices 创建设备 | 接收JSON请求体，验证字段，创建设备记录 | 创建成功返回201+设备信息；字段验证失败返回422；重复MAC返回409 |
| P1.2 | 查询设备API | GET /api/devices 查询设备列表 | 支持分页/过滤/排序参数 | 返回设备列表+分页信息；过滤条件正确生效 |
| P1.3 | 设备详情API | GET /api/devices/{mac} 查询单个设备 | 以MAC为路径参数查询设备详情 | 存在返回200+设备信息；不存在返回404 |
| P1.4 | 更新设备API | PUT /api/devices/{mac} 更新设备 | 接收JSON请求体，更新设备信息 | 更新成功返回200+更新后信息；不存在返回404；字段验证失败返回422 |
| P1.5 | 删除设备API | DELETE /api/devices/{mac} 删除设备 | 以MAC为路径参数删除设备 | 删除成功返回204；不存在返回404；在线设备删除返回409 |
| P1.6 | 设备状态查询API | GET /api/devices/{mac}/status 查询设备状态 | 返回设备当前在线/离线状态和最后心跳时间 | 返回状态准确（与心跳检测结果一致）；包含最后心跳时间 |
| P1.7 | 设备配置API | GET/PUT /api/devices/{mac}/config 读写设备配置 | GET读取配置，PUT写入配置 | GET返回完整配置；PUT写入后从设备读取确认变更生效 |
| P2 | 流控制API | 流启动/停止、转码控制、分发管理 | | |
| P2.1 | 启动流API | POST /api/streams/{device_mac}/start 启动视频流 | 验证设备存在且在线，启动拉流进程 | 启动成功返回200+流信息（URL/编码/分辨率）；设备离线返回409；已在流返回409 |
| P2.2 | 停止流API | POST /api/streams/{device_mac}/stop 停止视频流 | 停止拉流进程，释放资源 | 停止成功返回200；未启动返回404 |
| P2.3 | 流状态API | GET /api/streams/{device_mac} 查询流状态 | 返回流的当前状态和元数据 | 返回状态准确（active/stopped/error）；包含码率/分辨率/帧率信息 |
| P2.4 | 流快照API | POST /api/streams/{device_mac}/snapshot 截取视频帧 | 从当前流中截取一帧 | 返回截图URL或base64图片；响应时间<2秒 |
| P3 | 校准API | 校准启动/停止、参数查询、结果获取 | | |
| P3.1 | 启动校准API | POST /api/calibration/{device_mac}/{type} 启动校准 | 验证设备存在，启动指定类型的校准流程 | 启动成功返回200+校准ID；设备离线返回409；校准进行中返回409 |
| P3.2 | 停止校准API | POST /api/calibration/{id}/stop 停止校准 | 停止正在进行的校准，回滚已修改参数 | 停止成功返回200；校准已完成返回404；参数正确回滚 |
| P3.3 | 校准进度API | GET /api/calibration/{id} 查询校准进度 | 返回校准当前步骤和状态 | 返回进度信息（当前步骤/总步骤/状态）；实时更新 |
| P3.4 | 校准结果API | GET /api/calibration/{id}/result 获取校准结果 | 返回校准完成后的结果报告 | 结果包含每项校准的通过/失败状态和具体数值 |
| P4 | 数据查询API | 观测数据查询、设备历史、统计聚合 | | |
| P4.1 | 观测数据查询API | GET /api/observations 查询观测数据 | 支持时间范围/设备/类型过滤，分页 | 返回观测数据列表；过滤条件正确生效 |
| P4.2 | 设备历史API | GET /api/devices/{mac}/history 查询设备历史 | 返回设备状态变化历史和配置变更历史 | 历史记录按时间排序；支持时间范围过滤 |
| P4.3 | 统计聚合API | GET /api/stats 获取统计信息 | 返回设备总数/在线数/流状态等统计 | 统计数据准确；响应时间<100ms |
| P5 | 认证与授权 | JWT认证、权限控制、API限流 | | |
| P5.1 | JWT认证 | 实现JWT token认证 | 登录接口返回JWT，后续请求携带token | token验证正确；过期token返回401；无效token返回401 |
| P5.2 | 权限控制 | 基于角色的权限控制 | 定义admin/operator/viewer角色，控制API访问 | 无权限访问返回403；权限检查在中间件层统一处理 |
| P5.3 | API限流 | 限制API调用频率 | 配置限流规则（如100次/分钟） | 超过限制返回429；限流按IP或token区分 |
| P6 | 错误处理 | 统一错误响应、异常拦截、日志记录 | | |
| P6.1 | 统一错误响应 | 标准化错误响应格式 | 所有错误返回统一格式{error: {code, message, details}} | 错误格式一致；包含错误码和人类可读信息 |
| P6.2 | 异常拦截 | 全局异常捕获和处理 | 捕获未处理异常，返回500+错误信息 | 未处理异常不泄露堆栈信息；错误信息记录日志 |
| P6.3 | 请求日志 | 记录所有API请求和响应 | 中间件记录请求方法/路径/耗时/响应状态 | 日志包含完整请求信息；敏感数据脱敏 |

| P7.1 | OpenAPI文档 | 自动生成API文档 | FastAPI自动生成/docs和/redoc | 文档完整准确；每个端点有描述和示例 |
| P7.2 | 集成测试 | 编写API集成测试 | 使用pytest+httpx测试所有端点 | 所有端点测试通过；覆盖正常和异常场景 |
