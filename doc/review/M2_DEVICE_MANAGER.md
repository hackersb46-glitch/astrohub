# M2 设备管理器

## 概述
多设备管理、设备状态监控、设备生命周期管理。

## 前置约束
> 版本: v1.0
> 文档用途: 每次运行后逐项自检，达成标准则该项通过，未达成则记录问题
> 开发过程中，前一个P没有通过，不得跳过当前P，必须修复直到通过评审点
> 不允许做任何假设，如有问题，就是评审点不通过
> 任何有明确返回错误代码的，都需要查询错误码具体是什么
> 必须全量搜索已有案例、查询官方文档，解决问题
> 遇到阻滞，第一时间回看官方文档，查找关键词，全量阅读。禁止向人类发问
> 每一个P自检通过评审之后，将正确方法记录在对应method.md表格的对应操作方法中

## 评审明细

| 编号 | 标题 | 内容描述 | 操作方法 | 评审标准 |
|------|------|---------|---------|---------|
| P0 | 设备列表管理 | 建立设备增删改查核心功能、MAC唯一标识、批量操作 | 实现设备CRUD接口，以MAC为唯一主键 | 能正确创建、读取、更新、删除设备记录，MAC唯一性约束生效 |
| P0.1 | 创建设备记录 | 以MAC为唯一标识创建设备记录 | 接收设备信息（MAC/IP/型号/凭证），验证MAC格式（12位十六进制，允许:-分隔符），写入持久化存储 | 写入后可通过MAC精确查询到完整设备信息；重复MAC创建时返回明确错误并阻止 |
| P0.2 | 查询设备列表 | 按条件过滤查询设备 | 支持空查询（返回全部）、按MAC精确匹配、按型号模糊匹配、按状态过滤 | 查询结果与存储数据完全一致；过滤条件正确生效；无匹配时返回空列表而非错误 |
| P0.3 | 更新设备信息 | 修改已有设备配置参数 | 以MAC为键，更新IP/凭证/备注字段，保留创建时间和MAC不变 | 更新后查询确认变更生效；修改MAC操作被拒绝并返回错误；更新不存在的MAC返回明确错误 |
| P0.4 | 删除设备 | 从设备列表移除指定设备 | 以MAC为键删除，删除前检查设备是否在线（在线设备拒绝删除） | 删除后查询确认设备不存在；在线设备删除操作被阻止；删除不影响其他设备记录 |
| P0.5 | 批量导入设备 | 从JSON/CSV批量导入设备列表 | 解析导入文件，逐条验证MAC格式和必填字段，批量写入 | 返回成功数/失败数/失败详情；部分失败时成功条目已写入；非法格式文件返回解析错误 |
| P0.6 | 批量导出设备 | 导出设备列表为JSON/CSV | 按指定格式导出全部或过滤后的设备列表 | 导出文件可被批量导入功能正确读取；数据完整无丢失 |
| P0.7 | MAC唯一性校验 | 确保MAC作为全局唯一标识 | 所有写入操作前检查MAC是否已存在 | 重复MAC写入被阻止；错误信息包含已存在MAC的设备详情 |
| P1 | 设备状态监控 | 在线/离线检测、心跳探测、异常告警 | | |
| P1.1 | 在线检测 | 检测设备当前在线状态 | 通过HTTP GET /ISAPI/System/deviceInfo 检测设备可达性，设置超时阈值（默认5秒） | 在线设备返回200+设备信息；离线设备在超时后返回不可达；连续3次检测确认状态变更 |
| P1.2 | 心跳探测 | 周期性发送心跳检测设备连接 | 每隔N秒（可配置，默认30秒）发送轻量请求，记录响应时间和状态 | 心跳超时（连续3次无响应）标记设备离线；心跳恢复后标记在线并记录恢复时间 |
| P1.3 | 心跳间隔可配置 | 心跳周期支持运行时调整 | 提供配置接口修改心跳间隔，立即生效 | 修改后下次心跳按新间隔执行；间隔范围限制（最小5秒，最大300秒） |
| P1.4 | 状态变更通知 | 设备状态变化时触发通知 | 维护设备状态缓存，检测到变化时触发回调或写入事件队列 | 在线→离线时触发通知，包含设备MAC/变更时间/原因；离线→在线时同样触发 |
| P1.5 | 异常分类告警 | 识别异常类型并分类告警 | 区分认证失败(401)/连接超时/设备故障(500)/网络不可达等类型 | 每种异常有对应告警级别（warning/error/critical）；告警信息包含设备MAC、异常类型、时间戳、详细描述 |
| P1.6 | 状态历史记录 | 记录设备状态变化历史 | 每次状态变更写入历史记录（时间戳+旧状态+新状态+原因） | 历史记录可按时间范围和设备查询；记录不可修改或删除 |
| P2 | 设备配置管理 | 配置文件读写、参数备份恢复、配置差异对比 | | |
| P2.1 | 读取设备配置 | 从设备ISAPI接口读取当前配置 | GET请求设备配置端点，解析返回的XML/JSON，转换为内部格式 | 读取的配置字段完整；与设备Web界面显示一致；读取失败时返回错误而非空数据 |
| P2.2 | 写入设备配置 | 修改设备配置参数 | PUT/POST请求写入配置到设备，验证写入结果 | 写入后从设备读取确认变更生效；写入失败时回滚并返回错误 |
| P2.3 | 配置备份 | 备份设备当前配置到本地文件 | 导出设备所有可配置参数为JSON文件，包含时间戳和设备MAC | 备份文件包含完整配置；文件名格式 config_{MAC}_{yyyyMMdd_HHmmss}.json |
| P2.4 | 配置恢复 | 从备份文件恢复设备配置 | 读取备份JSON，逐项写入设备配置，验证恢复结果 | 恢复后设备配置与备份时完全一致；不支持的参数跳过并记录警告；恢复失败时设备保持原配置 |
| P2.5 | 配置差异对比 | 对比两份设备配置的差异 | 递归对比两份配置JSON，逐字段比对，输出差异报告 | 差异报告列出所有不一致字段，标注字段名、旧值、新值；完全相同时返回"无差异" |
| P2.6 | 配置版本管理 | 维护配置修改历史版本 | 每次配置变更自动创建版本快照，支持版本号查询和回滚 | 可按版本号查询历史配置；回滚后设备配置恢复到指定版本 |
| P3 | 设备分组 | 按类型/位置分组、批量配置、分组操作 | | |
| P3.1 | 创建分组 | 创建新的设备分组 | 指定分组名称和描述，创建分组记录，名称唯一性校验 | 分组创建成功；同名分组创建被阻止 |
| P3.2 | 设备加入分组 | 将设备加入指定分组 | 以MAC和分组ID为键，更新分组-设备关联关系 | 操作后查询分组设备列表确认结果；设备不可重复加入同一分组；不存在的设备/分组返回错误 |
| P3.3 | 设备移出分组 | 将设备从分组中移除 | 移除分组-设备关联，不影响设备本身 | 移除后设备不在该分组列表中；设备记录本身保留 |
| P3.4 | 删除分组 | 删除分组及其关联关系 | 删除分组记录，解除所有设备关联 | 分组和关联关系全部删除；设备记录不受影响 |
| P3.5 | 分组批量操作 | 对分组内所有设备执行操作 | 遍历分组内设备，逐台执行（状态查询/配置下发等），汇总结果 | 返回操作汇总（成功数/失败数/失败详情）；分组为空时返回明确提示 |
| P4 | 设备日志 | 操作日志记录、日志查询、日志导出 | | |
| P4.1 | 操作日志记录 | 记录设备管理的完整操作日志 | 每次操作记录[info][warning][error][done][failed]级别+时间戳(毫秒)+操作详情+操作者 | 日志格式统一；追加模式不覆盖；日志文件自动轮转（按大小或日期） |
| P4.2 | 日志查询 | 按条件过滤查询日志 | 支持按时间范围/级别/设备/操作类型过滤，支持分页 | 查询结果与过滤条件完全匹配；分页参数正确（页码/每页条数/总条数） |
| P4.3 | 日志导出 | 导出日志为文件 | 导出为CSV/MD/JSON格式，包含完整操作历史 | 导出文件可被标准工具打开；数据完整无丢失；大文件导出时分块处理不卡顿 |
| P4.4 | 日志清理 | 定期清理过期日志 | 按保留天数配置（默认30天），清理过期日志文件 | 清理后只保留指定天数内的日志；清理操作本身记录日志 |
| P5 | 设备生命周期 | 激活/停用/删除状态流转、状态持久化 | | |
| P5.1 | 设备激活 | 新设备首次录入系统 | 完成发现→认证→能力探测→保存凭证全流程后标记状态为active | 激活后设备状态为active，可正常执行所有操作；激活流程不可跳过任何步骤 |
| P5.2 | 设备停用 | 停用设备但保留配置数据 | 标记设备为inactive，停止心跳探测和状态监控 | 停用后设备不参与分组操作和自动发现；配置和日志数据完整保留 |
| P5.3 | 设备重新激活 | 已停用设备重新激活 | 从inactive状态恢复到active，重新启动心跳和监控 | 重新激活后设备恢复正常工作状态；状态历史记录包含完整的激活/停用轨迹 |
| P5.4 | 设备删除 | 永久删除设备及其所有数据 | 确认设备为inactive状态后，删除设备记录、配置、日志、分组关联 | 删除后所有相关数据均不存在；active状态设备删除被阻止；删除操作记录在审计日志中 |
| P5.5 | 状态流转约束 | 定义并 enforcing 状态流转规则 | 定义 valid transitions: new→active→inactive→deleted | 非法状态流转被阻止（如 active→deleted 必须先 inactive）；状态流转记录在日志中 |
