### Pan无限位速度计算方法

**原理**：Pan无限位时，位置范围0-3600循环，跨越边界时发生跳变。

**计算步骤**：

1. 检测跳变点：相邻两点差值 > 1800
2. 分段计算距离（绝对值）：
   - 跳变前：从起点到边界（0），距离 = abs(起点 - 0)
   - 跳变后：从边界（3600）到终点，距离 = abs(3600 - 终点)
3. 累加各段距离

**示例**：
- 起点：3590
- 终点：100
- 跳变次数：1

计算：
- 第一段：abs(3590 - 0) = 3590
- 第二段：abs(3600 - 100) = 3500
- 总距离：3590 + 3500 = 7090

### Tilt停止逻辑

1. 达到限位20点持续 tmin/tmax

两个条件满足任意一个就停止采样。

### CSV路径机制

**流程**：
1. `run_all_tests` 调用 `get_data_path_write(model_short, mac_clean, 'speed')`
2. 每次创建新序号目录（如 `20260530_001`）
3. CSV 在测试开始时以 "w" 模式创建（写表头）
4. `measure_speed_single` 以 "a" 模式追加数据
5. `post_process_csv` 使用当前测试的 CSV
6. 结果保存到 `speed.json`，包含 `csv_path` 字段

**不会多次测试混在同一文档**：每次测试创建新目录。

## Python 字节码缓存问题

### 现象
- 修改 `.py` 文件后，函数行为不变
- `inspect.getsource()` 显示新代码
- 但实际执行旧逻辑

### 根因
- Python 缓存 `.pyc` 字节码文件
- 文件修改后，解释器可能仍用缓存的旧字节码

### 解决方法
1. 添加 `print` 语句强制文件修改时间更新，触发重编译
2. 删除 `__pycache__` 目录
3. 重启 Python 进程

### 教训
- 如果代码修改后行为不变，检查字节码缓存
- 调试时添加 `print` 是有效的强制重编译方法
