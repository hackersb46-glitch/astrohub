## v6.13 异步加载修复

### 问题根源
老板反馈：白平衡、快门、光圈刷新后不加载，已修复4次仍然无效。

**真正原因**：`loadDeviceImageParams` 中 `apiGet('/function')` 和后续 `loadShutter/loadIris` 是**并行执行**，不是顺序执行！

```javascript
// 错误写法（之前）：
apiGet('/function').then(...);  // 异步，不等待
loadShutter(deviceIp);          // 立即执行，deviceFunctionData 还未加载

// 正确写法（现在）：
apiGet('/function').then(function(r) {
    deviceFunctionData = r.data;  // 先加载完成
    loadShutter(deviceIp);        // 再调用依赖函数
});
```

### 关键教训
1. **JavaScript Promise 不阻塞**：`.then()` 后的代码立即执行，不会等待
2. **依赖数据必须嵌套在 .then() 内**：需要 `deviceFunctionData` 的函数必须在 Promise 成功回调内调用
3. **相同问题反复出现 = 根本原因未找到**：不要只看表面调用，要看执行顺序

### 其他修复
- OSD 开关状态同步：`updatePtzOsdButtons/updateInfoOsdButtons` 添加开关状态更新
- 降噪提示：`setNoiseReduce` 添加 `showToast` 反馈
- WASM 状态：`clickStartRealPlay` 成功后调用 `updateWasmStatus('connected')`

## v6.36
- **问题**：WASM 视频流无法播放，所有静态文件返回 404
- **根本原因**：静态文件挂载路径错误
  - 挂载 `/static` → `src/web/`
  - 实际静态文件在 `src/web/static/`
- **修复**：`main.py` 第 253 行
  ```python
  app.mount("/static", StaticFiles(directory=str(_WEB_DIR / "static"), html=True), name="static")
  ```

## 教训
1. **静态文件挂载**：FastAPI `StaticFiles` 挂载时，路径必须精确指向静态文件所在目录，而不是其父目录
2. **UI 状态同步**：初始化完成后必须调用状态更新函数同步 UI
