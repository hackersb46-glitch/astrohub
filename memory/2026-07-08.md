## 任务3：互锁修复+定时预检查（17:42 ~ 19:23）

### 互锁竞态条件（已批准，待实施）
**现状**：
- `_search_state = {'active': None, 'lock': asyncio.Lock()}`
- `_guarded_search()` 中 `active` 的读写没有锁保护
- 竞态条件：多个请求同时检查 `active is None` 并同时设置

**修复方案**（已批准）：
- 使用 `asyncio.Lock()` 保护 `_search_state['active']` 的读写
- 先到先得（依赖 FastAPI 请求到达顺序）

### 定时触发前检查基线（已批准，待实施）
**方案**（已批准）：
1. `region_base.py` 新增基线管理函数：
   - `read_search_baseline(device_ip, mode)` → 读取基线
   - `write_search_baseline(device_ip, mode, baseline)` → 写入基线
   - 存储位置：`data/search_baseline.json`

2. `router.py` 预检查逻辑：
   - 定时触发：在 search 端点开头调用 `_check_baseline()`
   - 手动框选：在 `_finishRegion()` 开头调用 `/api/v1/vision/check-baseline`
   - 框选完成：在 Searcher 的 done 事件中写入基线

3. 阈值：
   - 对焦（反差）：15%
   - 白平衡（delta）：10%
   - 亮度（亮度值）：5%

4. 预检查流程：
   ```
   截图 → 计算当前值 → 读取基线 → 计算差异% → 
   if 差异 < 阈值: 返回"无需调整"
   if 差异 >= 阈值: 继续执行搜索
   ```

5. 基线存储：
   ```json
   {
     "<device_ip>": {
       "focus": {"contrast": 120.5, "timestamp": "2026-07-08T12:00:00"},
       "wb": {"red": 95, "blue": 102, "delta": 0.05, "timestamp": "..."},
       "brightness": {"brightness": 65.0, "shutter": "1/25", "iris": "F1.6", "gain": 45, "timestamp": "..."}
     }
   }
   ```
