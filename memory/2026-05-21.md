## 2026-05-21 WASM 登录调试

### 关键发现：HTTP Session 一致性

**问题根因**：WASM SDK sessionLogin 认证需要 HTTP session 一致性
- GET /ISAPI/Security/sessionLogin/capabilities → 摄像头生成 sessionID + challenge
- POST /ISAPI/Security/sessionLogin → 摄像头验证（需要同一 TCP 连接/session）

代理每次创建新 aiohttp.ClientSession，摄像头无法跟踪认证状态 → 401 Unauthorized

### 解决方案

修改 ISAPI 代理使用 per-device 持久 session：
```python
_isapi_sessions = {}  # device_ip -> aiohttp.ClientSession

# 在代理处理中
session = _isapi_sessions.get(device_ip)
if not session or session.closed:
    session = aiohttp.ClientSession()
    _isapi_sessions[device_ip] = session
```
