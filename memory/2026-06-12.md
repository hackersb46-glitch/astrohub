# 2026-06-12 Memory

## v6.10 数据存储路径重构

### 核心修改
1. **路径统一**: 所有设备数据迁移到 `data/devices/{mac}/` 目录
2. **MAC 格式**: 统一为无分隔符小写（如 `240f9b764193`）
3. **删除旧文件**: `ptz_config.json`、`device_config.json`

### 修改文件
- `src/config_paths.py` - 添加 DEVICES_DIR、REGISTRY_FILE
- `src/core/ptz_manager.py` - ConfigManager 使用新路径
- `src/core/device_manager.py` - DeviceManager 使用新路径
- `src/core/startup.py` - 保存设备到 devices/{mac}/info.json
- `src/advanced/device_path.py` - 统一路径函数
- `src/advanced/function.py` - 输出到动态路径
- `src/main/main.py` - 修复 get_module 问题
- `src/api/router.py` - `_resolve_device_id_to_ip` 支持无分隔符 MAC

### 关键教训
1. **_normalize_mac 统一**: 必须全局一致，无分隔符小写避免目录名不匹配
2. **Orchestrator 不是模块管理器**: main.py 直接创建 PTZManager 实例
3. **热重载问题**: 使用 --reload 时 _managers 可能未正确初始化
4. **router.py MAC 格式**: `_resolve_device_id_to_ip` 需同时支持冒号、横杠、无分隔符格式

### 数据结构
```
data/
├── devices/
│   └── {mac}/
│       ├── info.json              # 设备信息
│       ├── function_{ts}.json     # 功能探测
│       ├── limit_{ts}.json        # 限位测试
│       └── speed_{ts}.json        # 速度测试
└── registry.json                  # 设备注册表
```
