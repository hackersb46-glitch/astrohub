### 发现的 Bug

#### Bug 1（核心）：R² fallback 不回退到 best_pos
- **位置**: autofocus.py `_fit_and_search()` R²<0.6 分支
- **问题**: R²<0.6 时直接返回 `"action": "stop"`，电机停在过冲位置
- **修复**: R²<0.6 时进入 APPROACH 状态，逐步回退到 best_pos
- **同样修复**: "样条拟合失败"和"搜索失败"两个 fallback 路径

#### Bug 2：平坦检测延迟
- **位置**: autofocus.py 第249行
- **问题**: `_step_count >= 20` 才检查平坦，浪费10步
- **修复**: `_step_count >= 20` → `_step_count >= 10`

#### Bug 3（已修复，上一轮）：样条振荡导致虚假峰值
- **修复**: 黄金分割搜索区间从全范围缩小到 `best_pos ± 2` 步

#### Bug 4（已修复，上一轮）：电机步长线性假设错误
- **修复**: 一次性长移动 → 离散步进回退（每步0.2s，与扫描一致）

#### Bug 5（已修复，上一轮）：低质量数据进入拟合
- **修复**: 拟合前添加 R²<0.6 检验

### 修复后的算法流程
```
1. SCAN: 收集数据点
2. 峰值确认 → _fit_and_search()
3. R²检验:
   - R²≥0.6: 三次样条 + 黄金分割（best_pos±2范围）
```

## 教训

### 教训1：算法 fallback 路径必须回退到 best_pos
- 所有 fallback（R²低、拟合失败、搜索失败）都不能直接 stop
- 必须计算回退步数，进入 APPROACH 状态
- 电机停在过冲位置 = 虚焦

### 教训2：电机步长非线性
- 0.2s × 3次离散移动 ≠ 0.6s 一次连续移动
- 电机加速/减速阶段固定，短脉冲与长移动效率不同
- 回退时必须用与扫描相同的步长（0.2s/步）

### 教训3：平坦检测要早
- 波动数据（上下抖动但整体平坦）不会触发连续5步下降
- _decline_count 反复清零，永远到不了5
- 平坦检测阈值应从20降到10
