## 2026-05-22 10:00 关键发现：szAuth 为空

### 问题根因

**SDK 设备对象 `szAuth` 属性为空！**

```
SDK STATE: {
    'szIP': '192.168.5.72',
    'szAuth': '',  // ← 空字符串！
    'bSupportWebsocket': True,
    'iWebSocketPort': 7681,
    'iWSPort': -1,
    'oStreamCapa': {...}
}
```

### szAuth 来源

1. **SDK V2 登录流程**:
   - `ISAPI/Security/sessionLogin/capabilities` → 获取 sessionID, challenge, salt
   - 计算密码 hash: `sha256(userName + salt + password)` 迭代 N 次
   - `ISAPI/Security/sessionLogin` → 登录获取 WebSession cookie
   - `ISAPI/Security/token` → **szAuth 从这里获取**

2. **问题**:
   - `/ISAPI/Security/token` 返回空字符串
   - 原因: 代理没有正确转发 WebSession cookie 到摄像头

### 直接测试摄像头 ISAPI

```python
# 登录成功后，摄像头返回 Set-Cookie: WebSession=xxx
# 后续请求必须携带此 cookie
resp = session.get(f"http://{ip}/ISAPI/Security/token?format=json")
# 返回: {"TokenInfo": {"szAuth": "xxx"}}
```

### 修复方向

**ISAPI 代理必须正确转发 Set-Cookie 和 Cookie**

1. `sessionLogin` 响应 → 返回 Set-Cookie 给浏览器
2. 后续 ISAPI 请求 → 转发浏览器的 Cookie 到摄像头

### 文件修改

- `main.py`: ISAPI 代理需要正确处理 Cookie 转发

### 测试验证

```python
# 直接访问摄像头登录成功后获取 token
curl -H "Cookie: WebSession=xxx" http://192.168.5.72/ISAPI/Security/token?format=json
# 返回: {"TokenInfo":{"szAuth":"..."}}
```

### 下一步

1. 修复 ISAPI 代理的 Cookie 转发
2. 确保登录响应 Set-Cookie 正确传递给浏览器
3. 确保后续请求 Cookie 正确转发到摄像头
