# 2026-05-29 开发日志

## 错误复盘与教训

### 错误1：硬编码命名（test_all）
**问题**：使用`DEVICE_ID = "test_all"`作为设备标识，不是真实设备MAC
**发现**：老板指出"test_all"不是设备真实标识
**教训**：
- 设备唯一标识必须是MAC地址，从ISAPI实时获取
- 禁止使用临时命名（test_all、default等）

### 错误2：过度开发（device_path.py）
**问题**：未经要求创建新模块
**发现**：老板问"device_path.py是什么东西？我要求你开发的吗"
**教训**：
- 禁止未经要求开发新功能
- 每个新模块必须先确认需求

### 错误3：路径规范不明确
**问题**：创建的路径逻辑无法回答"单独运行测试时如何选择目录"
**发现**：老板问"每个脚本读取前一个的逻辑是什么，有多个json怎么选择"
**教训**：
- 设计前必须考虑完整运行场景
- 必须明确读取规则（匹配MAC+型号，找最新目录）
- 文件命名必须包含序号（YYYYMMDD_001）

### 错误4：文件修改失败多次
**问题**：edit工具多次匹配失败，无法找到精确文本
**发现**：文件编码或格式问题导致文本匹配困难
**教训**：
- 大范围修改时考虑用Python脚本直接处理
- 修改后必须验证结果

### 错误5：回答包含乱码（英文残留）
**问题**：回复中混入不相关的英文内容
**发现**：老板指出回复异常
**教训**：
- 回复必须精简、连贯
- 禁止复制粘贴残留内容

### 错误6：记不住文件路径
**问题**：开发文档路径记不住
**发现**：老板问"文档放在哪的？"
**教训**：
- 开发文档路径：`astrohub/documents/`（大写命名：SPEED_TEST.md）
- Review文档路径：`astrohub/doc/review/`（M1_speed_method.csv）

### 错误7：不记录老板要求的标准
**问题**：老板说了10遍以上的规范，我不记录
**发现**：每次都要重新问
**教训**：
- 所有老板要求必须立即写入文档
- 文档路径：`documents/SPEED_TEST.md`

### 错误8：假设问题而非查数据
**问题**：用"可能""假设"回答问题
**发现**：老板多次纠正："我禁止你做假设"
**教训**：
- 必须查实际数据回答
- 禁止用"可能""假设"

### 错误9：测试顺序错误
**问题**：同方向走3次→换方向走3次
**发现**：老板要求正向反向交替重复3次
**教训**：
- 测试顺序：`for repeat → for direction`（正向反向交替）

### 错误10：速度计算时间写死
**问题**：`valid_time = len(valid) * 0.1`
**发现**：实际时间不是11秒
**教训**：
- 必须用时间戳计算实际时间

## 文件路径规范（最终）

### 路径结构
```
data/{短型号}_{mac_clean}/{YYYYMMDD}_{序号}/
    ├── function.json
    ├── limit.json
    └── speed.json
```

### 命名规则
- 短型号：设备型号前8字符，去除末尾数字（iDS-2DF8C）
- MAC地址：无冒号小写（240f9b764193）
- 序号：当天第几个测试（001, 002...）

### 读取规则
1. 根据MAC和型号匹配设备目录
2. 列出所有{YYYYMMDD}_{序号}子目录
3. 按日期和序号排序
4. 选择最新目录

### 写入规则
1. 当天已有记录 → 序号+1
2. 当天无记录 → 序号=001
