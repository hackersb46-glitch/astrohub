## WASM 视频流修复进展 (02:40)

### 根因链
1. **aiohttp ClientSession 吃掉 Set-Cookie** — ISAPI代理透传时，`Set-Cookie: WebSession=...` 被aiohttp存到cookie_jar，不暴露在headers中 → 浏览器收不到session cookie
2. **SDK szAuth为空** — V2登录流程中 `x()` 函数用同步XHR获取token，但现代浏览器弃用同步XHR → `success`回调不执行 → `szAuth=""`
3. **WS认证失败** — `szAuth`为空 → WebSocket连接不带认证 → 摄像机返回 `badAuthorization`

### 已修复
- ISAPI代理：从aiohttp cookie_jar提取cookie，手动添加Set-Cookie header到响应 ✅
- WebSocket代理：支持二进制数据，从cookie动态路由 ✅
- 路由注册：/ws, /, /{path:path} 三个WS路由 ✅

### 当前阻塞
- `index.html` JS语法错误：插入的 `continueAfterLogin` 代码导致 `}, 500);` 括号不匹配
  - 行1982: `setTimeout(() => { ... }, 500);` 正确关闭
  - 行2006: `function continueAfterLogin() { ... }, 500);` — 多余的 `, 500` 导致语法错误
  - 整个script块解析失败 → `clickLogin2` 不存在

### 修复方案
把 `continueAfterLogin` 的 `}, 500);` 改为 `}`，因为 `setTimeout` 已经在行1982正确关闭了。
