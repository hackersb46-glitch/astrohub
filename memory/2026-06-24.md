## v7.113 静默移动录像文件（WASM录制→服务端移动）

### 背景
- `uploadRecordedFile()` 弹窗是WASM SDK方案的天生限制：`I_StartRecord`存到浏览器下载目录，浏览器安全策略禁止JS静默读本地文件
- 删掉弹窗后文件留在Downloads，不进入配置路径
- 老板要求：不弹窗，静默保存到配置存储路径

### 方案
```
I_StartRecord → WASM→Downloads/日期/录像.mp4
I_StopRecord → fetch POST /api/v1/files/move-recording {filename, device_ip}
后端: shutil.move(Downloads, 配置存储路径)
✅ 不弹窗，不依赖浏览器API
```

### 为什么可行
- WASM SDK不提供setSavePath（无插件版砍掉了，ActiveX版才支持）
- ISAPI原生录制：设备无内置存储（hddList空），虽然start/stop返回200但无介质
- 浏览器File System Access API：仍需用户手势
- 服务端move：同机Python有文件系统权限，文件名是我们自己传的（精确已知）

### 改动
1. `src/api/router.py`：+45行，新增 `POST /api/v1/files/move-recording`
   - 复用 `_resolve_storage_path()` 获取配置路径
   - 搜索 `~/Downloads/日期/*.mp4`，匹配文件名后 `shutil.move`
2. `src/web/index.html`：`uploadRecordedFile()` → `moveRecordedFile()`
   - 弹窗→fetch POST 静默调用
   - 两处调用点都改完了（`consoleRecordStop`和`_doRecordStopAndRestart`）

### ISAPI 调研结论
- 设备（iDS-2DF8C832IXS-A）无内置HDD/SD卡，NAS支持NFS/SMB/CIFS但未挂载
- ContentMgmt/record端点：GET返回404但PUT的start/stop返回200
- ContentMgmt/search：返回403，不支持搜索录像文件
- 文档：WebSDK v3.4编程指南（34页）、ISAPI 2.0 IPMD Service（285页）全文已读
- IPMD文档不含ContentMgmt/record端点——该端点属于NVR/DVR文档

### WASM SDK API清单
- I_StartRecord/I_StopRecord：可用，存浏览器下载目录
- I_RecordSearch/I_StartDownloadRecord：搜索/下载设备端录像（需设备有存储）
- I2_CapturePic + cbCallback：可获取Uint8Array静默上传
- setSavePath/SaveDir：不存在（无插件版已砍）
