## v8.13 白平衡修复（2026-07-03 凌晨）

### 修复1：premature convergence bug
- **根因**: `actual >= best_delta * 0.70` 在每次刚更新 best_delta 后恒成立 → no_improve 白+1 → 粗调1-2步就升阶段
- **修复**: 对比更新前的 `prev_best`（仅在新 best 确认后才检查改善）

### 修复2：R/B 独立调整
- **根因**: 双通道联动导致 fine 阶段在最佳解附近振荡（R+1 B-1 → R-1 B+1 循环）
- **修复**: 每步评估 [双通道, 仅R, 仅B] 三个候选，选 delta 最小

### 修复3：阶段最小步数
- 粗调(±5): 最少6步, 70%容差
- 中调(±2): 最少3步, 80%容差  
- 精调(±1): 收敛检测, 95%容差, 最多5步强制停止
- 追踪 `best_delta` 全局最佳，stop 时回滚

## v8.26 对焦算法重写（14:30）

### 老板指令
- 去掉所有过度开发的阈值判断（0.7、Phase 0/1/2/3等）
- 核心逻辑：追踪连续上升/下降趋势≥4点 → 高斯拟合
- best 只用于验证：拟合峰值预测值 > best + 10% → 对焦失败 → 重试
- 3次失败 → 退出报错
- 平台检测：10步以上 + 波动<10% + 反差<100 → 非正常退出

### 老板否决的设计
- ❌ Phase 0/1/2/3/4 状态机（过度开发）
- ❌ `contrast < best * 0.7` 阈值（拍脑袋，无依据）
- ❌ rising≥2 + falling≥3 计数器（拍脑袋，无依据）
- ❌ `_phase2_best` 局部峰值追踪（仍是不必要的复杂度）

### 老板纠正的核心思路
- 不要试图判断"过了峰值"这个事件
- 而是：出现连续趋势≥4点 → 直接拟合
- 拟合后验证：预测峰值比采样最佳值高10%以上 → 说明没采到真正的峰 → 重试
- best 不是用来做中间判断的，只用于最终验证
