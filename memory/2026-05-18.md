## WASM SDK 登录 — 失败 ❌
- **状态**：`I_Login` → `sessionLogin` → 401 Unauthorized（通过代理）
- **根因链**：
  1. `git checkout -- src/main.py` 丢失了 ISAPI 代理路由代码 → POST 被 SPA fallback 拦截 → 405
  2. 代理使用 `aiohttp.ClientSession()` 临时会话 → cookies 不保持 → 摄像头无法关联 capabilities GET 和 sessionLogin POST
  3. 需使用持久化 `aiohttp.ClientSession` 带 `CookieJar(unsafe=True)` 保持 session
- **当前状态**：`main.py` 被 git checkout 还原到无 ISAPI 代理版本，需要重建
- **正确凭证**：admin / Nftw1357
- **正确代理逻辑**：
  - 注入 `BasicAuth` 传输层认证
  - 剥离 incoming `Authorization` 头（避免 aiohttp 冲突）
  - 保留 `Content-Length`（POST body 必须完整）
  - 剥离 `host`, `origin`, `referer`, `cookie` 头
  - 使用持久化 session 维持 cookies

## 教训
- **禁止 `git checkout -- src/main.py`** — 会丢失所有积累的代理修复
- 修复代码后必须先验证语法再启动服务器
- 临时 session vs 持久化 session 的区别是 WASM 登录 401 的核心
