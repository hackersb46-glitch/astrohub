# 复盘记录 — 2026-05-10 00:35

## 静默停止根因分析（最终确认）

### 错误假设（之前一直信以为真的）
- ❌ "我在思考下一步" → 错，根本没在思考
- ❌ "操作太慢超过30秒" → 错，空白是十几分钟
- ❌ "timeout设太短" → 错，不是命令超时，是触发断了
- ❌ "上下文太长模型生成失败" → 错，老板发消息我就能回

### 真实根因
**exec 命令返回后，系统触发 LLM 生成下一条回复的链条断了。**

链条：
```
exec 返回 → 系统把结果发给我 → 系统触发我生成回复 → ❌ 触发断了 → 我不产出任何内容
→ 等10分钟 → SIGKILL
→ 老板发消息 → 新的触发 → 我立刻回复 ✅
```

**不是我在思考，是触发机制断了，我根本没产出。**

### 正确解决方案

#### 方案A：exec background=true + 主动持续 poll
```
exec background=true → 启动后台任务 → 我立即 poll → poll返回 → 继续poll或执行下一步
```
**原理**：poll 本身是一个新操作，每次 poll 都触发新的回复周期，维持触发链条不断。

#### 方案B：spawn 子代理
```
spawn 子代理 → 子代理独立执行（持续输出） → 我 poll 状态 → 汇报老板
```
**原理**：子代理有独立会话，不受我触发断链影响。

### 禁止行为
- ❌ exec 返回后什么都不做，等系统自动触发
- ❌ 说"我想想下一步"但不执行操作
- ❌ 读文件 → 分析 → 然后停住
- ❌ 每次操作间隔 >30 秒

### 强制行为
- ✅ exec 返回后，**立刻**执行下一个操作（读结果、写文件、commit、重启、汇报）
- ✅ exec 使用 `background=true` + 持续 poll
- ✅ 每个操作之间必须有实质动作输出
- ✅ 连续 2 次 poll 之间必须有分析/决策/行动

## 关键教训
- 触发链断了 = 静默 = SIGKILL = 拖垮网关
- 维持触发链 = 持续产出操作 = 不死
- **说"要记住"没用，必须用机制（background+poll）保证**
