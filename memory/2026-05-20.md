## WASM SDK 登录调试进展

### 问题根因
1. **sessionIDVersion=2 处理 bug**: SDK I_Login 回调中 `if(i>2)...else if(i<2)...else` 结构，当 i===2 时走 else 分支（已确认有此分支）
2. **this 指向 bug**: 回调函数内部 `this.I_LoginV2` 指向错误，SDK 开头已定义 `f=this`，应使用 `f.I_LoginV2`

### 已修复
- `webVideoCtrl.js`: 回调中 `this.I_LoginV2` 改为 `f.I_LoginV2`, `this.I_LoginV1` 改为 `f.I_LoginV1`
- ISAPI 代理: 曾尝试将 sessionIDVersion 从 2 改为 3（后恢复原值，因 SDK 有 else 分支处理）
