### 关键教训（SOUL.md 级别）
1. **修改前审计**: 第一步永远是列出 `<script>` / `</script>` 和括号平衡，再动手
2. **3 次规则**: 同一文件增量修改超过 3 次 → 立即放弃，从备份一次性重建
3. **幂等脚本**: 修复脚本执行前检查目标是否已存在，避免重复
4. **Playwright 降级**: Windows headless 不稳定 → 优先 curl/静态审计 → 最后才 Playwright
5. **Python 文件**: 逻辑 > 3 行 → 写入 `.py` 文件，不用 `-c` 内联

### 验证流程（标准化）
```
1. Python 静态审计: 版本计数、脚本结构、括号平衡
2. grep 残留: 检查删除函数名是否归零
3. HTTP 验证: curl http://127.0.0.1:10280/ → 检查 200 + HTML 完整性
4. 浏览器手动验证（老板）: 导航切换、按钮交互
5. Playwright 仅用于最终回归（可选）
```
