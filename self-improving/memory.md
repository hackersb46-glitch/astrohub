# self-improving memory

## HOT Layer

### 核心行为规则
1. **所有代码修改直接执行** — 发现问题 → 用 edit/write 工具直接修改 → 验证
2. **任何测试逻辑必须先完整读取CSV原文** — 列D方法+列E标准，不得自己推断或创造
3. **E2E测试失败时先截图+分析HTML结构** — 确认选择器正确性后再重试，不要盲目试错
4. **精确修复，禁止打补丁** — 发现问题必须找到根因，不是在表面打补丁
5. **删除字段/变量前必须检查下游依赖** — grep 全项目，确认无引用后再删除（v7.113 教训）
6. **修改共享机制前必须全量搜索调用点** — grep 全项目画出调用拓扑，逐点修改，不假设只有一条路径（ERR-20260708-001）
7. **新增代码块时必须检查模块导入** — 添加新函数前，先 `read` 文件顶部导入部分，确认所有依赖模块已导入，用 `python -c "from src.xxx import yyy"` 快速验证（ERR-20260708-002）
8. **重启服务前必须确保旧进程已杀死** — 使用 `Get-Process python` 确认无残留，或 `netstat -ano | findstr :10280` 确认端口已释放（ERR-20260708-003）

### 精调算法铁律（v8.37 教训）⚠️ CRITICAL
1. **方向决策基于反差变化** — `contrast < best2` 立即反向，不等2步
2. **best3独立追踪最高反差** — best3 >= best2 是必须条件
3. **best1<100直接失败** — 不进入精调，场景反差不足
4. **移动后0.3s等待** — 对焦和白平衡都需要，确保设备稳定
5. **粗调平坦处理** — 20步平坦→反向→反向也平坦→0.5s加速

### 设备存储铁律（v7.17 教训）
1. **设备标识必须使用 MAC 地址** — 禁止使用 IP 作为标识
2. **MAC 格式统一**：无分隔符小写（如 `240f9b764193`）
3. **SADP 发现必定返回有效 MAC** — 设备无 MAC 无法入网
4. **存储路径**：`data/devices/{mac_clean}/`
5. **禁止重复存储** — 同一设备只能有一个目录（按 MAC）

### 禁止过度开发（v7.17 教训）
1. **老板说禁止过度开发** = 停止开发，只做精简检查
2. **不要新增功能** = 只修复问题
3. **不要扩展修改范围** = 只改老板明确要求的

### 备份三步法（v7.97 教训）⚠️ CRITICAL
1. **创建**: 执行备份命令
2. **验证**: `Test-Path` + 文件大小/数量确认
3. **记录**: 写入 memory 文件（路径、内容、时间）
- ❌ 不验证 = 没做备份

### 版本号铁律（v7.97 教训）
1. **constants.py 是唯一定义** — `VERSION` + `VERSION_NUM`
2. **HEARTBEAT.md 从 constants.py 同步** — 不手动写版本号
3. **每次代码修改后检查版本一致性**

### WASM 与 ISAPI 完全独立（v7.97 教训）
1. **PTZ 控制 → ISAPI** — 不依赖 WASM
2. **视频播放 → WASM** — 需要 WebSocket 代理
3. **可安全删除 WASM 而不影响 PTZ**
4. **设备凭据从 device_config.py 读取** — 不硬编码 IP/密码

### 搜索策略（v7.97 教训）
1. **全量搜索优先** — 先 `Get-ChildItem -Recurse`，不要逐轮试路径
2. **一轮到位** — 不要多轮猜测

### 工作流
- 发现问题 → 直接修复 → 验证 → 汇报结果
- 被老板纠正 → 立即写入 corrections.md
- 任务完成 → 验证修改 → 汇报结果
- 备份操作 → 创建+验证+记录，三步缺一不可
