# 纠正记录

## ERR-20260615-001: 设备存储使用 IP 作为 MAC

**时间**: 2026-06-15 04:18  
**错误**: `save_credentials` 和 `register_device` 用 IP 作为 MAC  
**后果**: 同一设备重复存储（按 IP 和按 MAC）  
**老板纠正**: "怎么可能出现没有 MAC 的 PTZ 设备？SADP 发现本身就能获取设备的 MAC、IP、网关"  
**根因**: 代码逻辑错误，允许无 MAC 注册  
**修复**: 拒绝无效 MAC，必须从 SADP 发现获取真实 MAC

## ERR-20260615-002: 在合并逻辑打补丁

**时间**: 2026-06-15 04:18  
**错误**: v7.16 在设备列表合并逻辑中按 IP 去重  
**后果**: 治标不治本，根本问题在存储逻辑  
**老板纠正**: "这个逻辑根本就是错误的！不是在这里打补丁修改，必须是精确修复问题"  
**根因**: 没有找到根本原因，在表面打补丁  
**修复**: v7.17 精确修复存储逻辑，删除错误存储
