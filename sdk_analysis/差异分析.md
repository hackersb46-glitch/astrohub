# 官方SDK vs AstroHub 码流切换差异分析

## 📌 核心发现

### 1️⃣ 官方SDK的码流切换模式（关键发现）

**官方demo.js中的正确模式**：
```javascript
function clickStartRealPlay(bProxy) {
    var oWndInfo = WebVideoCtrl.I_GetWindowStatus(g_iWndIndex);
    
    var startRealPlay = function () {
        WebVideoCtrl.I_StartRealPlay(szDeviceIdentify, {
            iRtspPort: iRtspPort,
            iStreamType: iStreamType,  // 主码流=1, 子码流=2
            iChannelID: iChannelID,
            bZeroChannel: bZeroChannel,
            bProxy: bProxy,
            success: function () { /* ... */ },
            error: function (status, xmlDoc) { /* ... */ }
        });
    };

    if (oWndInfo != null) {  // 已经在播放，先停止
        WebVideoCtrl.I_Stop({
            success: function () {
                startRealPlay();  // 停止完成后才启动新流
            }
        });
    } else {
        startRealPlay();  // 没在播放，直接启动
    }
}
```

**关键点**：
1. ✅ 先检查窗口状态 `I_GetWindowStatus()`
2. ✅ 只停止当前窗口 `I_Stop()`，不是 `I_StopAll()`
3. ✅ 用 **success回调** 等待停止完成，不是 `setTimeout`
4. ✅ `I_StartRealPlay` 在 `success` 回调中调用

### 2️⃣ Proxy模式 URL路径对比

#### 官方Nginx配置
```nginx
location ^~ /webSocketVideoCtrlProxy {
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    if ($http_cookie ~ "webVideoCtrlProxyWs=(.+)") {
        proxy_pass http://$cookie_webVideoCtrlProxyWs/?$args;
        break;
    }
}
```

**关键发现**：
- URL路径：`/webSocketVideoCtrlProxy` （**无channel路径参数**）
- Channel信息通过Cookie传递：`webVideoCtrlProxyWsChannel=102`
- `proxy_pass` 后加 `/?$args`，不添加路径

#### AstroHub的Python proxy (stream_wasm.py v7.87)
```python
# v7.87: 修复URL格式转换逻辑
if channel:
    ch_num = int(channel)  # 102
    ch_id = ch_num // 100  # 1
    st_type = ch_num % 100  # 2
    channel_path = f"/{ch_id}/{st_type}"  # /1/2
    wasm_log.info(f"WS: URL format converted: /{channel} -> {channel_path}")
```

**问题**：
- ❌ 添加了 `/1/2` 路径，但官方不需要
- ❌ Cookie中已经有channel信息，URL路径不需要重复

### 3️⃣ JSPlugin的URL解析逻辑

根据 `jsPlugin-3.0.0.min.js` 中的 `playCmd` 函数：
```javascript
playCmd(e, t, n) {
    t = {sequence: 0, cmd: t || "realplay", url: e};
    n && n.protocolType && (t.protocolType = n.protocolType);
    return JSON.stringify(t);
}
```

**JSPlugin接收的参数**：
```javascript
JS_Play(url, auth, wndIndex)  // 只有3个参数！
```

**JSPlugin如何解析URL**：
- 官方：`ws://192.168.5.72:7681/webSocketVideoCtrlProxy?version=0.1&token=xxx&...`
- JSPlugin从 **Cookie** 获取 channel信息（`webVideoCtrlProxyWsChannel=102`）
- **不从URL路径解析**（因为URL中没有channel路径）

### 4️⃣ Cookie设置时机对比

#### 官方SDK
```javascript
// 在I_StartRealPlay之后，SDK内部设置
document.cookie = 'webVideoCtrlProxyWs=192.168.5.72:7681; path=/';
document.cookie = 'webVideoCtrlProxyWsChannel=102; path=/';
```

#### AstroHub (当前实现)
```javascript
// clickLogin2中提前设置（登录前）
var channelNum = 100 + g_iStreamType;
document.cookie = 'webVideoCtrlProxyWsChannel=' + channelNum + '; path=/';
document.cookie = 'webVideoCtrlProxyWs=' + szIP + ':' + g_iWebSocketPort + '; path=/';
```

**问题**：
- ❌ 我们在登录前就设置Cookie
- ❌ SDK可能在登录后重新设置Cookie，覆盖我们的值
- ✅ 应该在登录成功后，由SDK内部设置Cookie

## 🎯 正确操作清单（根据官方SDK）

### ✅ 码流切换正确流程

```javascript
function changeStreamType(streamType) {
    var parsedType = parseInt(streamType, 10);
    g_iStreamType = parsedType;
    
    // 官方模式：先停止当前窗口
    var oWndInfo = WebVideoCtrl.I_GetWindowStatus(g_iWndIndex);
    
    if (oWndInfo != null) {
        // 窗口正在播放，先停止
        WebVideoCtrl.I_Stop({
            success: function () {
                // 停止完成后，启动新流
                clickStartRealPlay({ iStreamType: parsedType });
            },
            error: function () {
                console.error('Stop failed, but starting new stream anyway');
                clickStartRealPlay({ iStreamType: parsedType });
            }
        });
    } else {
        // 没在播放，直接启动
        clickStartRealPlay({ iStreamType: parsedType });
    }
}
```

### ✅ Proxy模式正确配置

**1. Cookie设置（由SDK内部处理，不需要手动设置）**
```
webVideoCtrlProxyWs=192.168.5.72:7681
webVideoCtrlProxyWsChannel=102
```

**2. URL格式（不需要添加channel路径）**
```
ws://192.168.5.72:7681/webSocketVideoCtrlProxy?version=0.1&token=xxx
```

**3. Python proxy转发规则**
```python
# 不需要转换URL路径
camera_ws_url = f"ws://{camera_ip}:{camera_port}{channel_path}?version=0.1&token={token}"
# 但 channel_path 应该是空字符串 ""，不是 "/1/2"
```

## 🛠️ 需要修复的问题

### 🔴 问题1：使用 `I_StopAll()` + `setTimeout`
**影响**：停止所有窗口，可能影响其他设备；`setTimeout` 不可靠

**修复**：改用 `I_Stop` + `success` 回调

### 🔴 问题2：Python proxy添加了 `/channel/streamType` 路径
**影响**：JSPlugin可能无法正确解析URL

**修复**：移除路径转换，保持 `channel_path = ""`

### 🔴 问题3：手动设置Cookie时机不当
**影响**：SDK可能覆盖我们的Cookie值

**修复**：移除手动Cookie设置，让SDK内部处理
